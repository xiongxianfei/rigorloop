## Purpose / big picture

Implement the approved first migration slice for single authored skill source handling. This plan removes the repository-local `.codex/skills/` mirror from tracked Git state, keeps it generatable from canonical `skills/`, preserves public adapter packages under `dist/adapters/`, and adds the validation/docs guardrails needed before later release-artifact migration work.

The plan intentionally does not remove public adapter skill copies. Public adapter release artifact packaging, compatibility-window release notes, and eventual `dist/adapters/**/skills` untracking require a later plan after this first slice is reviewed and implemented.

## Context and orientation

Canonical authored skill files live under `skills/<skill>/SKILL.md`. The local Codex mirror currently lives under `.codex/skills/` and is a byte-for-byte generated mirror built by `scripts/build-skills.py`. Public adapter packages live under `dist/adapters/` and are generated by `scripts/build-adapters.py` through `scripts/adapter_distribution.py`.

M1 changed `scripts/build-skills.py --check` to validate generated local mirror temp output instead of comparing against tracked `.codex/skills/` files.

M2 removes `.codex/skills/**` from tracked Git state, adds `.codex/skills/` to `.gitignore`, and updates contributor guidance. While this plan is active, `dist/adapters/**` remains tracked generated public adapter output and existing adapter drift checks remain in force.

## Non-goals

- Do not untrack, delete, or restructure public adapter skill copies under `dist/adapters/`.
- Do not change public adapter repository-tree installation paths in this slice.
- Do not generate, validate, or publish adapter release archives in this slice.
- Do not add release artifact metadata under `docs/reports/adapter-artifacts/releases/<version>.yaml` until a release publishes generated adapter archives.
- Do not change skill behavior or skill wording.
- Do not rewrite Git history to remove previously committed generated skill copies.
- Do not remove support for Codex, Claude Code, or opencode adapters.
- Do not replace public adapter drift checks while public adapter skill copies remain tracked.
